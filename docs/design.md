# Design System and UX Direction

## Overall design direction
The product should feel like a premium editorial profile, not a generic SaaS dashboard.

The mood is:
- calm
- intelligent
- restrained
- high-trust
- senior
- quietly technical

This is not a neon AI toy. It is a polished recruiter-facing experience.

## Brand positioning
David Robertson is presented as:
- Senior AI Architect
- Applied AI Systems Builder
- Product-minded engineer
- Solutions-oriented technical leader

The interface should reinforce clarity, rigour, and taste.

## Visual references
The tone can take inspiration from premium editorial and portfolio systems:
- spacious layouts
- serif headline typography
- muted colours
- soft cards
- subtle borders
- strong hierarchy
- minimal chrome

## Colour direction
Base palette should be warm, neutral, and premium.

### Primary palette
- Background: warm ivory / stone
- Text: charcoal / graphite
- Secondary text: muted slate
- Accent: muted amber / bronze
- Interactive highlights: soft slate-amber blend
- Cards: ivory with subtle border
- Borders: low-contrast warm grey

### Avoid
- bright blue SaaS defaults
- aggressive gradients
- neon AI visuals
- heavy shadows
- pure black UI
- oversaturated accents

## Typography
Use a premium serif for major headings and a clean sans-serif for body/interface.

### Recommended fonts
- Headings: Newsreader
- Body/UI: Manrope

## Typography behaviour
- Large serif headlines
- Clean sans body copy
- Generous line-height
- Tight visual hierarchy
- Strong spacing rhythm

## Layout principles
- Wide negative space
- Content centred with deliberate breathing room
- Minimal top navigation
- Sticky input at bottom of chat screen
- Soft cards for prompts and evidence
- No cramped dashboard density

## Chat page UX
The chat page should open with a calm empty state.

### Empty state components
- Back to profile link
- Name and role at top
- Centre icon or monogram block
- Large heading
- Short descriptive intro
- Four to six recruiter-focused prompt cards
- Large rounded message input pinned to bottom

### In-chat state
- User messages are simple and clean
- Assistant messages stream in progressively
- Source chips appear beneath answers
- Follow-up prompts appear after responses
- Subtle loader while stream begins

## Component styling rules

### Prompt cards
- Rounded corners
- Low-contrast border
- Soft background
- No loud hover effects
- Slight lift or border change on hover only

### Chat input
- Large rounded container
- Left icon
- Clean placeholder text
- Send action on right
- High readability
- Fixed to bottom on desktop and mobile

### Messages
- Assistant messages should feel editorial, not bubble-heavy
- Avoid cartoon chat bubbles if possible
- Prefer clean text blocks with role labelling
- Keep spacing generous

### Source chips
- Small, subtle pills
- Show evidence category, not internal IDs
- Examples:
  - Skills
  - Project: CareersAI
  - Experience
  - AI Systems
  - Preferred Roles

## Motion
Use motion sparingly.

### Allowed motion
- fade in
- slight upward easing
- subtle skeleton loading
- progressive stream reveal

### Avoid
- bouncing
- flashy animated gradients
- large-scale transitions
- gimmicky AI effects

## Content tone
All copy should sound:
- senior
- direct
- grounded
- useful
- recruiter-aware

Avoid hype language.

## Homepage direction
Homepage should present:
- name
- positioning
- short summary
- key proof points
- featured builds / projects
- CTA to chat
- CTA to view CV / projects / contact

## Chat intro copy
Suggested pattern:

Ask about David Robertson

Ask about David’s AI systems work, architecture background, technical stack, project delivery, and role fit. You can also paste a job description for a structured fit analysis.

## Suggested prompt examples
- What kinds of AI roles is David best suited for?
- What production AI systems has David built?
- What is David strongest at technically?
- Has David led full-stack AI product builds?
- What experience does David have with Python and Next.js?
- Paste a job description for a fit analysis

## Accessibility
- Strong text contrast
- Clear focus states
- Keyboard accessible prompt cards
- Sensible font sizes on mobile
- No information only conveyed by colour

## Mobile behaviour
- Sticky input must remain usable
- Prompt cards should stack cleanly
- Header must stay compact
- Long answers must remain readable with comfortable padding

## Final design standard
The site should feel like:
a premium editorial CV meets a carefully designed product demo.

It should not feel like:
a chat widget dropped into a portfolio.
