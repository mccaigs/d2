# Design System and UX Direction

## Overall design direction
Bidworx should feel like an operational procurement intelligence platform.

It should feel:
- dark
- precise
- controlled
- evidence-led
- command-infrastructure oriented

It should not feel like a generic chatbot, personal showcase, or legacy assistant.

## Brand positioning
Bidworx is:
- evidence-backed bid intelligence
- deterministic tender analysis
- compliance and risk infrastructure
- a source-backed procurement decision tool

## Colour direction

- background: `#07111F`
- surface: `#0D1B2E`
- surfaceAlt: `#101F35`
- primary: `#38BDF8`
- secondary: `#60A5FA`
- accent: `#22C55E`
- warning: `#F59E0B`
- danger: `#EF4444`
- text: `#F8FAFC`
- muted: `#94A3B8`
- border: `#1E3A5F`

## Typography
Use clean operational sans-serif typography. Avoid editorial personal-site styling.

## Homepage direction
Homepage sections:
- Hero
- Evidence-backed bid intelligence
- What Bidworx analyses
- Deterministic scoring
- Compliance and risk checks
- Tender intelligence workflow
- CTA to analyse a tender

## Chat page UX
The chat page should open with:
- Ask Bidworx
- Prompt cards:
  - Analyse this tender opportunity
  - What are the likely compliance risks?
  - What evidence do we need to support this claim?
  - Summarise the buyer requirements
  - Score this opportunity
  - Identify missing submission requirements
- Streaming answer area
- Evidence/source chips
- Bid readiness panel for pasted tender text

## Component styling
- Surfaces use `#0D1B2E` and `#101F35`
- Borders use `#1E3A5F`
- Primary actions use `#38BDF8`
- Positive/readiness signals use `#22C55E`
- Compliance warnings use `#F59E0B`
- Risks use `#EF4444`
- Cards should be square or low-radius operational panels

## Content tone
Copy should be:
- direct
- procurement-aware
- evidence-led
- operational
- careful about support and confidence

Avoid hype and unsupported claims.
